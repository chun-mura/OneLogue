# TODO & Time Tracker 仕様書

## 1. プロジェクト概要
ローカル環境（Docker）で動作する、TODO管理とタイムトラッキングを統合した自己管理ツール。計測したデータを可視化し、時間の使い方の改善を支援する。

## 2. 技術スタック
* **フロントエンド:** Next.js (App Router), Tailwind CSS, Recharts (グラフ描画)
* **バックエンド:** FastAPI (Python 3.11+), SQLAlchemy (ORM)
* **データベース:** SQLite 3 (ホストOSにボリュームマウントして永続化)
* **インフラ:** Docker, Docker Compose

## 3. 機能要件

### 3.1. タスク管理機能
* タスクの登録（名称、カテゴリ、優先度 [高・中・低]）
* タスクのステータス管理（未着手、完了、アーカイブ）
* タスクの編集・削除

### 3.2. タイムトラッキング機能（ストップウォッチ）
* **排他計測:** 一度に「開始」できるタスクは1つのみ。別のタスクを開始すると、実行中のタスクは自動で停止（記録保存）される。
* **リアルタイム表示:** フロントエンドで経過時間をカウントアップ表示。
* **履歴保存:** 「開始日時」と「終了日時」を1つのレコードとして保存。

### 3.3. ダッシュボード・分析機能
* **基本集計:** 指定期間（日・週・月・任意）のカテゴリ別合計作業時間。
* **改善インサイト:**
    * **集中度分析:** 1回あたりの平均計測時間（短い計測が多いカテゴリは中断が多い可能性）。
    * **時間帯別分布:** どの時間帯（朝・昼・夜）にどのカテゴリの作業が多いかのヒートマップ。
    * **優先度乖離:** 高優先度タスクに十分な時間を割けているかのチェック。

## 4. データモデル (ERD案)

### `tasks` テーブル
* `id` (PK, Integer)
* `title` (String)
* `category` (String) - e.g., "Work", "Hobby", "Personal"
* `priority` (Integer) - 1: Low, 2: Medium, 3: High
* `status` (String) - "pending", "completed"
* `created_at` (DateTime)

### `time_entries` テーブル
* `id` (PK, Integer)
* `task_id` (FK to tasks.id)
* `start_time` (DateTime)
* `end_time` (DateTime, Nullable) - 計測中はNull

## 5. アーキテクチャ構成

### 5.1. Docker構成
* **`frontend` サービス:** Port 3000
* **`backend` サービス:** Port 8000
* **`db_volume`:** `./data/db.sqlite3` をバックエンドコンテナにマウント

### 5.2. 主要APIエンドポイント (FastAPI)
* `POST /tasks`: タスク作成
* `GET /tasks`: タスク一覧取得
* `POST /tasks/{id}/start`: 計測開始（他を停止するロジック含む）
* `POST /tasks/{id}/stop`: 計測停止
* `GET /stats/summary`: ダッシュボード用集計データ取得

## 6. UI/UX イメージ
1.  **メイン画面:** 左側にTODOリスト（優先度順）、右側に「現在実行中のタスク」の大きなタイマー。
2.  **ダッシュボード画面:** * 上部に「今週の総作業時間」などのカード型メトリクス。
    * 中央にカテゴリ別の円グラフと、日次の積み上げ棒グラフ。
    * 下部に「改善のヒント（例：今週は趣味の時間が仕事より30%多いです）」などのテキスト分析。
